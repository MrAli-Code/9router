سیستم **9Router** یک مسیردهاب (Router) هوشمند و متن‌باز برای کدهای هوش مصنوعی است. این سیستم به شما اجازه می‌دهد ابزارهایی مانند Cursor، Claude Code، GitHub Copilot و غیره را به ارائه‌دهندگان رایگان (مدل‌های Claude، GPT و Gemini) متصل کنید و با مکانیزم‌هایی مانند RTK، مصرف توکن را ۲۰ تا ۴۰ درصد کاهش دهید.  
در ادامه، راهنمای **جامع، بسیار ساده و قدم‌به‌قدم** نصب، راه‌اندازی و عیب‌یابی این سیستم برای ۳ محیط مختلف (سیستم شخصی، سرور لینوکسی و هاست اشتراکی) گردآوری شده است.


### **پیش‌نیازهای کلی**

پیش از هر کاری مطمئن شوید موارد زیر در محیط شما نصب هستند:

* **Node.js**: ترجیحاً نسخه ۱۸ یا بالاتر (node \-v)  
* **مدیریت پکیج**: npm  
* **Git**: برای دریافت سورس کد پروژه


### **۱. نصب در سیستم محلی (Localhost / سیستم شخصی)**

اگر می‌خواهید این سیستم را روی لپ‌تاپ یا کامپیوتر شخصی خود (ویندوز، مک یا لینوکس) اجرا کنید، دو روش دارید:


#### **روش اول: نصب سریع سراسری (ساده‌ترین روش \- پیشنهادی)**

ترمینال (یا CMD/PowerShell) خود را باز کرده و دستورات زیر را وارد کنید:

Bash  
\# نصب پکیج به صورت سراسری در سیستم  
npm install \-g 9router

\# اجرای برنامه  
9router

🎉 **نتیجه:** داشبورد مدیریت در مرورگر شما در آدرس http://localhost:20128 در دسترس خواهد بود.


#### **روش دوم: نصب از طریق سورس کد (مناسب توسعه‌دهندگان)**

Bash  
\# دانلود کدهای پروژه  
git clone https://github.com/decolua/9router.git  
cd 9router

\# کپی کردن فایل متغیرهای محیطی  
cp .env.example .env

\# نصب پیش‌نیازها  
npm install

\# اجرای پروژه در حالت توسعه  
PORT=20128 NEXT\_PUBLIC\_BASE\_URL=http://localhost:20128 npm run dev

### **۲. نصب و راه‌اندازی در سرور (VPS / Dedicated Server)**

برای سرورهای لینوکسی (مانند اوبونتو)، برای اینکه برنامه با بستن ترمینال قطع نشود، باید از ابزارهای پایداری مانند **Docker** یا **PM2** استفاده کنید.

#### **روش اول: استفاده از Docker (بسیار پایدار و پیشنهادی)**

اگر روی سرور داکر نصب دارید:

Bash  
\# دریافت کدها  
git clone https://github.com/decolua/9router.git  
cd 9router

\# راه‌اندازی کانتینر در پس‌زمینه  
docker-compose up \-d

پروژه روی پورت 20128 بالا می‌آید و از طریق http://YOUR\_SERVER\_IP:20128 در دسترس خواهد بود.



#### **روش دوم: استفاده از PM2 (بدون داکر)**

اگر نمی‌خواهید از داکر استفاده کنید:

Bash  
\# نصب کنترل‌کننده PM2  
npm install \-g pm2

\# دریافت پروژه و نصب پیش‌نیازها  
git clone https://github.com/decolua/9router.git  
cd 9router  
cp .env.example .env  
npm install

\# بیلد کردن پروژه برای حالت پروداکشن  
npm run build

\# اجرای پایدار پروژه با PM2  
PORT=20128 HOSTNAME=0.0.0.0 NEXT\_PUBLIC\_BASE\_URL=http://YOUR\_SERVER\_IP:20128 pm2 start npm \--name "9router" \-- run start

\# ذخیره تنظیمات برای اجرای خودکار بعد از ریستارت سرور  
pm2 save  
pm2 startup

### **۳. نصب در هاست اشتراکی (cPanel / DirectAdmin)**

**نکته مهم:** این پروژه یک وب‌اپلیکیشنِ Node.js و یک روترِ همزمان است. به دلیل مصرف منابع (پردازش و رم) و نیاز به باز بودن پورت‌ها، **هاست اشتراکی محیط ایده‌آلی برای این کار نیست**. با این حال اگر هاست شما ابزار **"Setup Node.js App"** را دارد، این مراحل را طی کنید:

1. وارد cPanel شوید و روی **Setup Node.js App** کلیک کنید.  
2. روی **Create Application** کلیک کنید.  
3. نسخه Node.js را روی **18 یا بالاتر** تنظیم کنید.  
4. مسیر روت اپلیکیشن (Application root) را مثلاً 9router\_app تعیین کنید.  
5. سورس کد گیتهاب را به صورت یک فایل ZIP دانلود کرده، در فایل‌منیجر هاست در پوشه 9router\_app آپلود و اکسترکت کنید.  
6. فایل .env.example را به .env تغییر نام دهید.  
7. در بخش تنظیمات Node.js، روی دکمه **Run NPM Install** کلیک کنید تا ماژول‌ها نصب شوند.  
8. در همان صفحه متغیر محیطی NEXT\_PUBLIC\_BASE\_URL را برابر با آدرس دامنه خود قرار دهید.  
9. برنامه را **Start** کنید. ترافیک دامنه شما مستقیماً به پروژه متصل خواهد شد.

### **۴. نحوه استفاده (اتصال ادیتور به 9Router)**

فرقی نمی‌کند سیستم را کجا نصب کرده‌اید؛ پس از اجرای موفق، وارد داشبورد خود (مثلا http://localhost:20128) شوید:

1. از منوی **Providers** ارائه‌دهنده مدنظر (مثل Kiro AI یا OpenCode Free) را متصل (Connect) کنید.  
2. تنظیمات زیر را در ادیتور خود (مانند Cursor، Cline یا Claude Code) وارد کنید:  
   * **Base URL / Endpoint:** http://localhost:20128/v1 (اگر روی سرور است، آی‌پی سرور را جایگزین کنید).  
   * **API Key:** کلیدی که داشبورد سیستم به شما داده است.  
   * **Model:** یکی از مدل‌های ارائه‌شده (مثلاً kr/claude-sonnet-4.5).

### **۵. راهنمای کامل عیب‌یابی (Troubleshooting)**

**مشکل ۱: خطای اشغال بودن پورت (Error: listen EADDRINUSE: address already in use :::20128)**

* **علت:** پورت 20128 قبلاً توسط برنامه دیگری (یا اجرای قبلی همین برنامه) اشغال شده است.  
* **راه‌حل:** اجرای قبلی را متوقف کنید یا برنامه را روی پورت دیگری اجرا کنید:  
  Bash  
  PORT=3000 NEXT\_PUBLIC\_BASE\_URL=http://localhost:3000 npm run dev

**مشکل ۲: خطا در نصب پیش‌نیازها (npm install ارور می‌دهد)**

* **علت:** نسخه Node.js شما بسیار پایین (مثلاً 14 یا 16\) است.  
* **راه‌حل:** با دستور node \-v نسخه را بررسی کنید و با ابزاری مثل nvm آن را به نسخه 18 یا 20 ارتقا دهید.

**مشکل ۳: سایت روی سرور بالا می‌آید اما در مرورگرِ سیستم محلی باز نمی‌شود (Timeout)**

* **علت:** فایروال سرور (VPS) دسترسی به پورت 20128 را مسدود کرده است.  
* **راه‌حل (سرور ابونتو):** پورت را در فایروال باز کنید:  
  Bash  
  sudo ufw allow 20128/tcp  
  sudo ufw reload

**مشکل ۴: هوش مصنوعی پاسخی نمی‌دهد (ارور در گرفتن پاسخ از مدل)**

* **علت:** اتصال به Provider برقرار نیست یا محدودیت توکن (Quota) ارائه‌دهنده‌ی رایگانی که انتخاب کرده‌اید تمام شده است.  
* **راه‌حل:** داشبورد سیستم را باز کنید. در بخش Providers چک کنید که وضعیت اتصال "سبز" باشد و موجودی شما صفر نشده باشد. سیستم به طور خودکار باید روی مدل‌های جایگزین Fallback کند اما گاهی نیاز به اتصال مجدد اکانت دارید.

**مشکل ۵: یافتن دلیل قطعی در سرور**

* **اگر با داکر نصب کرده‌اید:** برای دیدن لاگ‌ها تایپ کنید:  
  Bash  
  docker-compose logs \-f

* **اگر با PM2 نصب کرده‌اید:** برای دیدن لاگ‌ها تایپ کنید:  
  Bash  
  pm2 logs 9router
